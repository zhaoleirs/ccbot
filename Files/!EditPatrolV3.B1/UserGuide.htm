Edit Patrol User Guide

 

 

Version 2

 

 

Topic

 

Here is a basic documentation concerning EditPatrol. The tool allows drawing or modifying your glider profile in WIZIWIG or reviewing it directly on a map. You can edit an existing profile or create it from scratch. The tool adds some extra tags in the XML in order to avoid selection of the continent during multiple sessions. EditPatrol allows also following remotely your character’s progression.

 

Import/Export Menu

 

The Import/export menu, you can only export the database of add-on. For some legal issues, I can distribute the build-in database. But you can easily reconstitute a full database by sharing your add-ons.

Edit Menu

 

The edit menu allows to copy and past waypoints. You can copy all waypoints. But for pasting, you need 2 conditions. If you have 1 waypoints selected, the past will insert the selection just after the selected point. If you 0 waypoints selected, the past will add at the end the selection to the current patrol. You have also in this menu various functions to manipulate the selection.

 

Menu View

The menu view allows selecting a continent, in Edit Patrol, coordinates are shared between the 3 continents. So, when you load an exiting profile you have to select the right contents where the profile has been recorded using this menu.

  

 

Map Generating

 

The setup of EditPatrol does not include WOW maps, because there is a legal issue and because it is more reliable. Therefore, you have to create maps using your installed version of WOW.  EditPatrol needs to know where is installed WOW.  It uses the registry key of WOW.  Therefore, Make sure that Wow is installed and run at least once in order the registry key is set. After this, edit patrol will be able to generate all maps.

 

The scan menu of the Map generator Dialog, allows checking if there are new patches available. The Patches are sorted by dates in order to get all latest resources.

 

Maps customization

By default, EditPatrol supports only few maps, the most interesting. But you can add more maps such as instance maps,  or raid maps. The first time you is loaded EditPatrol, the tool create a file including all parameters mandatory identifying a map. The name of this file is “MapAddOn.ini”.  You can edit this file and add your own maps using the following syntax.

 

After the update of this file, you must recreate maps using the “Creating maps..”.

 

Each map is identified with an ID [Map_(X)] that identifies the order in the menu, for example [Map_9] will place the map at the 9em position.

Create a New profile

 

To create a new profile, you should first use the menu “new” and select the continent in the selection dialog. To create the first Waypoint you have to use the contextual menu or the accelerator F2 or F1. After this first point, you can continue to draw your path using ENTER.

The “ENTER” key duplicates and moves the waypoint you have selected.  It is also a way to insert waypoint in an existing patrol. If you select a waypoint, ENTER create a new waypoints if you select a “ghost waypoint”. ENTER will create ghost waypoint just after the previous one.

 

The best practice is to create the first profile using glider, you can use a horse or protection casts in order do not engage a fight and defines a raw patrol.  This allows defining a patrol and being sure, you will not block in an obstacle. Then, in a second time, you can edit your patrol, and modify the patrol in order to be optimal. The Monster database will be very helpful for that.

 

Contextual Menu

 

You can apply some action on the selection of waypoints, or apply tools using the contextual menu on the map. To show the contextual menu, you have to click on the right button of the mouse when the mouse is focused on the map.  If a menu is grayed, it is mean the action is not possible on the objects you have selected. For example, if you want to add a monster to the “avoid list “, you have first to move the mouse on top of a monster icon and then click on the right button.

Navigation

 

To navigate in the map, you can use the wheel’s button of the mouse. The wheel zooms and translates the position in the map.  There is some short cut to navigate without the wheel button. The keyboard arrows allow panning the map, the keys Prev and Next allows zooming out and zooming in.

 

Selection and Edition

 

The left button allows selecting on waypoint (+ shifting to keep other selected waypoints). 

You can make a multi-selection of waypoints, by keep pushed the left button of the mouse and drags the mouse to make a selection rectangle. All points in the rectangle will be selected.

 

To move a single or a group a waypoints, click on a selected point (yellow scare) and move freely the group on the screen.

To switch between Waypoints or Ghost Waypoints, uses the Combox on top of the list box.

The Lisbox shows the Wow waypoints

 

 

 

Item Database

 

Edit patrol includes an Item database, the item’s location are not very precise. It is just an indication for the user. However, it is very useful to create a patrol for farming or to update a profile in order to be optimum. It is also useful to avoid monster very dangerous

Edit patrol  allows transferring an item from a database to another For example, the US database is very light comparing the FR. Using this tool, allows you to update US database from the FR only by translating the monster’s name, and all occurrences of an item will be copied and will be translated in same time. The tool creates a new database including all new updates called xxx_addon.bin. This database is loaded in addition of the original one. You can go back to the original database, by deleting this file.

 

 

Radar Panel

 

The radar allows monitoring glider running in a second PC and connected to the network. You have just to enable the remote control option on glider and change the port and password if you want. Enter these parameters on the edit patrol in the radar panel. Enter in the “remote address”the IP address of your Glider Pc. You can put the address in any form “www.xxxxx.com” “192.0.168.8” or simple the PC name if you are in a local network “MYSECONDPC”. You can test this option using a single PC and using the 127.0.0.1 address, but it is quite slow because Edit Patrol and WOW run on the same machine.

After this, once glider is running, you should be able to connect glider. On the map, you can see the position of your character as a green dot.

 

If you validate the echolocation, edit patrol will draw an animation like a radar echo.

If you validate “auto center “edit patrol will follow your character during its glide. The value “View range” allows setting the zoom of the map. As soon as you quit the Radar panel, the link will be broken.

You can also see the screen shots if you check the “capture” check box

when the panel Radar is selected, you can optionally activate the MiniView from the contextual menu of the map.  This allows reducing the Windows size and places this window in “top most” this option is cool to do something else on your pc and see where your character is. A double click on the Map allows to quickly switching from miniview and full screen.



During Gliding, you can see on the caption of the top most windows the state of glider, “Disconnected”, “Stopped”, Fighting, Alive, etc...”.

 

 The interface “Telnet” allows to quickly changing the Glider Behavior. Stop the Glider, Restart etc…

 

The Auto Add-on check box allows updating your monster database using the remote interface of glider. During a fight, Edit Patrol analyses the /status and plot the monster’s location on the map in a private database. When you leave the session, the private data base is merged with the current add-on file. You can export and add-on file using the menu “monster add-on”. You can also update and add the add-on file by merging other XML files using the “Import” Menu.

The auto-Log check box allows logging the real patrol of your character. You can see directly on the screen and on the autolog.xml the path including unsolicited trajectory due to a fight. You can reload en edit patrol the file autolog.xml. You will see the exact trajectory of your character during the session. When you load the autolog.xml, you will see in a tool tip of each waypoint the current status of your character when he reaches the selected point.   For performances consideration, the number of waypoints displayed is limited, but you can set the number of waypoints displayed on the screen using the persistent waypoints edit box. You can change the color of the log-path by dbl-click on the color box.

 

 

 

 

 

Auto update

 

Edit patrol includes an auto update system. This tool allows being always up to date with item database and the program. In the preference menu, you can also activate the automatic check. It is not set by default.  This option allows checking the presence of a new version, each time you run Editpatrol.  But you can also check for a new version manually, using the button “ Update Now”.

 

Profile Panel

 

This panel allows setup the profile.  For all option you can set in the panel, please refer to the Glider manual.

 

The Add Faction and Compute Faction from Path, this feature is not working yet. The feature is visible but not active in a Glider Session. This feature requires a reversing of the Wow database or a modification of Glider by the use of the Tag <FactionName> in the profile XML.

Option Panel

 

This panel allows setup the behavior of EditPatrol

 

“My Level” allows to set your character level, if you set your character level, edit patrol compute on the map all monsters you can kill easily or not, each monster has a color, yellow is ok, red is hard to kill for your character like in WOW. You have to type enter to validate your choice.  

 

Sometime it is difficult to see the path of the patrol on the map. Therefore, you can change the color of the path waypoints by double clicking on the color control and select the new color in the dialog box.

 

There are two check boxes, which allow disabling the display of the waypoints and ghost waypoints. This is useful when the ghost waypoints and waypoints share the same path.

 

 

 

“Print selection on the map” allows printing the type of the item directly on the screen.  This prints only the item you have selected in the listbox. This option is very useful when you look for a graveyard or a taxi somewhere close to your patrol.

 

“Filter Item Name” allows showing only items with a specific keyword. For example if you want harvesting Mageroyals and Peaceblooms. Enter in the filter edit control “Mageroyal Peacebloom”. It will be easy now to make a patrol for farming specifics set of herbs.

 

Group Panel

 

This panel is mainly used to ease defining a group profile. You can define 4 profiles in the 4 slots. These profiles will be rendered on the render view, but you can’t edit them. You can show and hide Group profiles using the show check box. You can change the transparency and the color using the slider control and the dblclick on the color box.
